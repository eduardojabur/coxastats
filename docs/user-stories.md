# User Stories

## Sprint 1

- Como **administrador**, quero cadastrar jogadores para manter o elenco atualizado.
- Como **administrador**, quero cadastrar times para organizar adversarios e elenco.
- Como **administrador**, quero cadastrar partidas com placar para consolidar resultados.
- Como **torcedor**, quero autenticar no sistema com permissao de leitura para acompanhar os dados com seguranca.

## Sprint 2

- Como **administrador**, quero editar jogadores em uma interface responsiva para operar o sistema no notebook ou no celular.
- Como **usuario**, quero receber mensagens claras de erro e sucesso para entender o resultado das minhas acoes.
- Como **torcedor**, quero consultar os dados pelo frontend sem poder alterar registros.

## Sprint 3

- Como **equipe de desenvolvimento**, queremos documentar requisitos e fluxos em BDD/UML para alinhar implementacao e avaliacao.
- Como **administrador**, quero enviar fotos dos jogadores para enriquecer o cadastro visual.
- Como **equipe**, queremos pipeline automatizada para validar o projeto a cada envio ao GitHub.

## Sprint 4

- Como **gestor esportivo**, quero um dashboard com medias, vitorias e gols por jogador para apoiar analises gerenciais.
- Como **usuario**, quero alternar idioma entre portugues e ingles para demonstrar suporte internacional.
- Como **gestor tecnico**, quero consultas SQL otimizadas para responder rapido mesmo com mais partidas registradas.

## Sprint 5

- Como **equipe**, queremos testes automatizados no backend e no frontend para reduzir regressao.
- Como **avaliador**, quero evidencias de cobertura e monitoramento para validar a qualidade tecnica da entrega.
- Como **operador**, quero health checks e validacoes finais para ter mais confianca na execucao do sistema.
